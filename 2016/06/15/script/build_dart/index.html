Build Dart-SDK

使用制定版本的 gcc

Build Dart-SDK 要求使用至少 gcc-4.6.4 编译，在某些环境下不太可能（公司机器及操作系统），方便的做法是自己编译一个 gcc-4.6.4 并全部使用 static-lirary

Build shared library VM

dart 提供了 standalone 版本的可执行程序，却没有提供 so 版本，如果希望嵌入 dartlang 脚本必须要将程序放置到 dart 的编译环境当中，另外一种方式是生成一个 vm.so，具体方法如下:

  • 为所有的编译选项增加 ‘cflags’: [‘-fPIC’] (runtime/bin/zlib.gyp, runtime/bin/bin.gypi, runtime/dart-runtime.gyp, runtime/vm/vm.gypi, third_party/boringssl/boringssl_dart.gyp, runtime/third_party/double-conversion/src/double-conversion.gypi)
  • 复制一个 dart 的target(runtime/bin/bin.gypi)将目标改成 shared_library 名称改为 dartvm,并在 sdk/dart.gyp 当中添加到 runtime 的目标下
  • 编译时可能会报 multiple definition, 在编译选项当中增加 -Wl,–allow-multiple-definition

Trouble Shooting

Some dart file not generated

有可能是我们使用的 libc 版本过低，最好编译一个高版本的 gcc 通过 LD_LIBRARY_PATH 制定 libc 和 libstdc++ 的路径，如果希望编译出的可执行程序不依赖于 libc 和 libstdc++ 则直接用静态版本的 gcc 编译即可，这两个选项不冲突

Old Version Linux

使用老版本的 Linux 的时候可能会报 GLIBC-2.15 not defined，这是因为旧版本的 linux (如CentOS6.X) 使用的 glibc, gcc 等比较老，不过 dart 似乎仅仅依赖于这个库，使用当前版本的 gcc 编译一个 libc-2.15.so 就可以了。

如果 gcc 过老连 libc-2.15.so 都编译不了就装一个新版本的 gcc 编译它就可以了，因为 glibc 是最底层的库，只在要本机编译基本就可以用了，不用担心他担心 libstdc++ 等库。

丢失了文件

编译时报错，某些文件找不到，不知道是不是没有 checkin, 不过 github 上却是有这个文件（HEAD），直接下载下来放好编译
