编译 libHDFS

使用 cdh 版本的 hadoop, 默认 libhdfs 不会编译

可能遇到的问题

  • cc1: error: unrecognized command line option “-m”

gcc 使用 -m 选项指定 target arch, 出现这个错误的原因是 -m 后面没有跟参数。在所有没有参数的 -m 后面增加 64 即可

  • include/stddef.h:211: error: duplicate ‘unsigned

这个错误是因为 Makefile 当中包含

编译 hdfs_test

hdfs_test 是个 c++ 访问 hdfs 的小程序，在初始阶段测试还是有一定用处的，但编译它经常出问题 ( make test )。还好他的依赖项非常少

libHDFS

文件随机写

HDFS 是不支持文件随机写的，为了支持随机写必须要将文件下载到本地操作完成后，再将文件上传至原始位置。为了避免其他文件访问正在写的文件（实际上是文件锁操作），还需要将文件挪到其他位置。

文件锁的另外一个方法是为文件创建一个 .lock 文件，表明其状态，如果这么做所有访问 hdfs 文件都必须通过一个支持这种规则的客户端访问才行。
