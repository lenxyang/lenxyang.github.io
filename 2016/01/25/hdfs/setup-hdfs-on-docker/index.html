Docker 配置静态 IP 比较麻烦，/etc/hosts 文件在启动 docker 时自动生成，之前的更改可能被覆盖，为 docker 集群提供一个 DNS 服务是个不错的方法，一个 DNS 缓存服务器就足够用了，完全不需要完整的服务（架设，配置都比较麻烦）。

Trouble Shooting

部署 dnsmasq 时出现错误 “setting capabilities failed: Operation not permitted”

具体解决办法，修改 /etc/dnsmaqs.conf

参考 Debian Bug report logs - #514214 dnsmasq crashes if fails to set capabilities

连接 ssh 服务是出现 “ssh Read from socket failed: Connection reset by peer”

docker image 可能是安装非常初步的操作系统，各种错误都用可能出现。这个问题是 sshd 异常中断导致的，可以使用 /sbin/sshd -d 启动程序，而后通过 ssh 连接服务，sshd 会打印足够的日志来诊断问题。
