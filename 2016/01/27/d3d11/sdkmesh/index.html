SDKMesh

Jan 27, 2016 | DirectX | Hits

Skinned Mesh

SDKMesh 的基本思路是：

  • 静态 mesh 是 bind pose 形态的 mesh
  • bone 的矩阵保存的是如何从各个组件挪移到 bind_pose 的矩阵 (需要传递依赖)
  • 顶点数据包含绑定的骨骼及其权重，但骨骼数据并非所有的骨骼而是起到作用的骨骼

Bone 格式

字段 matrix 是 bind pose 矩阵，他是相对于父节点的，如果想要完整的计算 bind pose 矩阵，还有通过 Hierarchy Transform。计算这个矩阵是必须的，它描述了如何将节点移动到坐标原点的变换（这个矩阵的逆）。

此外，anim_index 是帮助 DXUT 实现 Skinned Mesh 而增加的，如果自己实现，可以不用这个字段。

Animation Transform

Influence Matrix

一般来说，动画数据并非所有的骨骼都会动。从上面的结构也可以看书，具体哪些骨骼移动了需要从 FrameName 当中查找，如果没有出现过就是没有移动的。动画所涉及的骨骼就是 Influence Matrix。这个概念很重要，因此在计算完骨骼动画插值以后需要进行 skinned mesh blending，每个顶点对应的 index 是 influence matrix 的 index, 而非 bone 的 index。

Influence Matrix 的存在原因在于每次设置每一帧都需要传递数据，通过Influence Matrix 可以减少 memory - GPU memory 的数据传输。不过对于游戏来说，这个机制看起来比较多余，对于哪些包含多组动画的 skinned mesh，很少会有不移动的 bone，这时候实际上 influence matrix 能够减少的 bone 的数量就很少了。

Animation Interval

确定帧

SDKMesh 是确定 FPS 的动画，而且帧数比较密集，只要确定帧直接渲染即可，不需要进行插值计算。确定帧的操作也很简单

Hierarchy Transform

每个骨骼的变换都会影响他的子节点的变化，因此在完成自身的变换以后还要将这个变化组合上父节点的变换传递给子节点，这个过程叫做 Hierarch Transform

Local Transform

默认情况下，skinned mesh 处于 bind pose 状态，每一个部分所在的位置并非其 Local Coordinate 的坐标原点，这时候所有的 translation, scaling 和 rotation 都是非法的。必须首先将他还原到坐标原点而后进行 transfrom，还原到坐标原点的操作可以通过 bind pose 的逆矩阵来完成。

Blending

Blending 的过程将定点所涉及的所有 Matrix 进行 blending 而后计算最终位置，这个过程既可以在 Vertex Stage 实现，也可以在 CPU 过程当中完成。
