Tessellation based method

Tessellation based 的方法能够不影响固有程序流程做到 decal, 调优方面的工作基本全部都在 hlsl 内部，这种方法有非常好的适用性。

此方法的流程如下

  • 确定 hit position: 找到与 mesh 匹配的点
  • 在 ds 当中将 decal texture 映射到 hit position 上
  • 而后计算 displacement

GCC Tricks

去掉 debug info

一般情况下 debug info 是不需要的，发布时可以包括两个版本带有调试信息和不带调试信息的，上线使用不带调试信息的，如果出现 core 在使用带有调试信息的追问题。（某些时候带宽真的是问题，发布的程序越小越好）

连接静态 libgcc, libstdc

libgcc 是非常基础的库，他基本与编译器是绑定的，如果部署的机器 gcc 版本较低，很有可能会因为 libgcc 的版本没办法运行

Direct3D11 Particle System Devnotes

2016-02-03

Particle

Notes

  • 用于 Stream Out 的 VertexBuffer 必须使用 kBindTargetStreamOut
  • 需要 3 个 VertexBuffer, 初始化和两个输入输出补丁切换的 VertexBuffer
  • VertexBuffer 的大小需要按照最多的粒子数指定 (初始化的 VertexBuffer 不需要)
  • 在绘制 VertexBuffer 时，需要将其从 StreamOut Stage 上解除绑定
  • 使用 DrawAuto，以便 DirectX 自行决定绘制的粒子数
  • 创建 DrawAuto 的 Geometry Shader 时，使用 CreateGeometryShaderWithStreamOut 创建 Geometry Shader

最后一点最为重要，网上的大部分 demo 都用到了微软 Technique 框架，通过框架编译代码通常会很简单。在对照写 c 代码时很有可能漏掉这个细节。

Trouble Shooting

展现一段时间后 Particle 的渲染对象消失了

猜测可能是因为 VertexBuffer 设置太小，导致 Emitt 例子无法加入其中导致消息，将 VertexBuffer 加大后问题解决。
