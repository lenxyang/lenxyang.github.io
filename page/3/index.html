Vertex Buffer Tricks

Multi-Stream VertexBuffer

Input Assemble 支持将多个 Vertex Buffer 当作同一个输入（通过 InputLayout 支持），这种操作有诸多好处

重复利用数据

以地形为例，顶点数据是变化较大，但是所有的地形的 texcoord 都是类似的，最多就是起始点不同。因此所有的地形 tile 都可以共享一个 texture vertex buffer。

Instanced Draw

通过 Instance 可以较少渲染 API 调用的数量，但每一个 instance 都有不同的参数设置，如果通过 shader 的 constant buffer 来获得，需要考虑最大长度等问题，还要通过下标访问获得。更简单的方法是通过一个独立的 Vertex Buffer (使用 Per Instance 模式）设置 instance 的参数，在 Input Assemble Stage 中，参数就随顶点一起传入到了 Vertex Stage 当中，更方便灵活。

Stencil Buffer

Mirror based on Stencil Buffer

基于 Stencil Buffer 的镜面效果的基本思想是：将所有可以被反射的物体在镜面映射的角度进行渲染，使用 Stencil Buffer 作为渲染的可接受范围（此范围实际就是镜子的范围）

基于 Stencil 的镜面映射基本步骤如下

  • 绘制基本场景
  • 清理 Stencil Buffer 状态，
  • 仅仅在 Stencil Buffer 上渲染镜子
    • 设置 stencil buffer 状态
    • 不写 depth buffer (否则镜子后面的世界没办法通过深度测试)
  • 在 stencil test passed 的情况下，渲染所有镜面映射的物体
    • 对比 stencil buffer 的状态，成功时才写入
  • 渲染镜子 (开启 blending)

设置反射物体的 StencilState

注意：一定要注意 stencilref 的设置 (SetDepthStencilState 的第二个参数)。第一次绘制镜子时他的作用实际就将 “留下标记”(stencilref 写入到 stencil buffer)，记录那里是镜子的区域；绘制反射物体的时候，采用了 kCompareFuncEqual 方法，之后当 stencilbuffer 的值与 stencilref 的值相同时，stencil test 才能成功，物体才能够绘制。

Picking 的步骤

Direct3D11 Detailmap

2016-01-27

Detailmap

Tessellation based method

Detailmap(Tessellation based) 能够在渲染管线级别上决定几何体的细节，比起 normalmap 和 parallex mapping 要真是的多。(normalmap 无法处理遮挡关系，如果细节起伏较大会失真; parallex mapping 一定程度上解决了遮挡关系的问题，但如果某个高度变化到不可见状态时，parallex mapping 是没办法处理的)

Tessellation based method 还有诸多优点

实现细节

Density Map

Tessellation 的分割参数 (Hull Constants Shader) 与很多因素相关，具体可以分成动态及静态两种；具体来说，与摄像头的距离是动态的，与摄像头的方向夹角这些因素是动态的；此外，Mesh 对象本身的特征是静态的，高度变化较为剧烈的地方应该采用较密集的分割，变化平缓的地方采用较少的分割。高度图是静态的，所以可以采用离线或者加载时刻生成的办法生成一张分割参数的 Texture，用来帮助 Tessellation 的 Hull Constants Shader 的性能

具体可以参考 CreateDensityMapFromHeightMap

svn-access-control

2016-01-26

Svn Access Control

目录级权限管理

修改文件 /etc/httpd/conf/httpd.conf， 增加配置项 AuthzSVNAccessFile

AuthzSVNAccessFile 文件内容

AuthzSVNAccessFile 的内容看起来是这样

通过 repo 文件管理权限

默认情况下 SVN 使用 DAS 方式管理权限直接配置在服务器端，不过这样做的坏处时权限管理必须访问 SVN 服务器才能够完成。

较为方便的方法是在 Repositiory 当中增加特殊文件表达权限管理，在 checkin 的时候（通过 svn 的 hook 机制）自动生成 DAS 权限管理文件

Trouble Shooting

Commit blocked by pre-commit (exit code 255) with no output

可能的原因包括

  • pre-commit 没有可执行权限
  • pre-commit 有语法错误

svn: E175013: Access to ‘https://100.85.23.45/svn/test1‘ forbidden

有可能 AuthzSVNAccessFile 的路径配置错误了， 查看 /var/log/httpd 的日志确定错误
